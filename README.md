## 4-1 Redux-thunk 이해하기
> 1.
>> redux의 기능을 향상시켜주는 middleware인 redux thunk를 사용할 것이다. (근데 실제 프로젝트에서는 saga를 쓸 것이다. 왜냐하면 saga가 기능이 더 많다.)
>> 비동기 액션을 할 수 있게 하는 것이다.
> 2.
>> thunk의 주요 기능 : 하나의 동기 액션 안에 여러가지 동기액션을 넣을 수 있다.
> 3.
>> thunk는 그냥 지연의 의미만 담은 프로그래밍적 단어이다.
> 4.
>> 원래 action은 request, success, failure 이렇게 3개씩 나눠야 한다.  

## 4-2 sage 설치하고 generator 이해하기
> 1.
>> function* 이 있는데 generator라는 문법이다.
> 2.
>> generator는 함수인데 안에 yield;(발음: 일드) 라는 것이 있다. 역활은 기준점같은 것임.
> 3. 
>> 실행하기 위해서는 generator.next()라는 것을 통해서 실행해야함. 그러면 yield를 기준으로 함수가 실행됨. 즉, 중단점이 있는 것임. 다 끝나면 done이 true가 됨.
> 4.
>> while(true)를 쓰면 절대 멈추지 않는 함수가 됨. (이 특성을 이용해서 쓰기도 함)

## 4-3 saga 이펙트 알아보기
> 1.
>> call은 동기함수 호출이다. fork는 비동기함수 호출이다.
>> 비동기는 요청 보내버리고 결과 기다리지 않고 바로 다음이 실행됨.
>> 동기는 요청 보내버리고 리턴값 기다림.
>> all은 밑에 다 실행하는 것.
>> put은 dispatch와 같은 것임.

## 4-4 take, take 시리즈, throttle 알아보기
> 1. 
>> take은 일회용임. 그래서 다음 호출할 때 사라져 버림.
>> while(true)로 감싸서 써야함. 
>> 근데 이렇게 안 쓰고 takeEvery로 이벤트리스너를 씀 
>> takeLatest는 버튼 두 번 눌렀을 때, 마지막 것만 실행함.
>> 첫번째꺼는 takeLeading이 있다.
>> 다만 치명적인 단점이 요청은 취소를 못하기 때문에 요청할 때 db에 어떤 정보를 저장한다면 2번 저장될 수도 있다. 
> 2.
>> 이런 경우에는 throttle이라는 것이 있다. 2초 이렇게 설정해 놓으면 2초 동안은 딱 한 번만 요청할 수 있다.
